! Data Source Principle (Critical)

🔵
✅ CORRECT — Real Orbit-1 Agent

Messages sent via WebSocket to production API. Agent responses captured live. Tagged orbit1_live. Dashboard shows "Real Agent Responses."

⚠️
FALLBACK — Offline Simulation

No API connection. Hardcoded/generated agent messages (not real). Tagged self_generated. Dashboard shows "Offline Simulation — Not Real Agent."

❌
NEVER — Self-Validation

The flow generator predicts what the agent MIGHT say. These are PLANS, not test data. Scoring self-generated messages is meaningless.

✦ Design Principles

1. Validate the Real Agent

Test the Ghaia Orbit-1 RFP Agent — not predictions.

  • Phase 0 sends real messages to Orbit-1 via WebSocket
  • Real responses tagged orbit1_live
  • Self-generated marked with ⚠️ banner

2. Governance-First

Every execution passes through governance gates. No validation can override a deterministic governance failure.

3. Strict Separation

  • Shared: contracts only — zero business logic
  • Workflow: domain state — no I/O
  • Governance: policy — no transport
  • Runtime: orchestration — delegates all logic

4. Preview/Run Integrity

AI Flows uses store-and-reuse: preview → store on client → run sends exact plan → no regeneration. Config change resets.

5. Event-Driven Communication

Modules communicate through EventBus — never direct imports. Enables loose coupling, easy testing, runtime observability.

6. Deterministic Override Guard

AI validator cannot upgrade scores downgraded by deterministic checks. deterministicOverrideGuard: true ensures governance failures are never masked.
